ANÁLISIS DEL PROYECTO EXISTENTE
Estado Actual del Repositorio (cac-validador)
El proyecto anterior es un validador básico de archivos planos para el reporte CAC. Tras el análisis del Instructivo PDF (CAC-IEP1-I01) y el repositorio original, se identifican los siguientes gaps y oportunidades de mejora:
Análisis del Instructivo CAC (Resolución 0247/2014)
El instructivo define 134 variables distribuidas en 168 campos. Se identificaron las siguientes agrupaciones críticas para el motor de validación:
Variables 1-16: Identificación EAPB y usuario (nombre, ID, fecha nacimiento, sexo, ocupación, régimen, municipio)
Variables 17-44: Diagnóstico, estadificación y antecedentes oncológicos (CIE-10, HER2, Gleason, TNM/FIGO)
Variables 45-73: Terapia sistémica primer y último esquema (quimioterapia, hormonoterapia, medicamentos ATC)
Variables 74-85: Cirugía (CUPS, IPS, fechas, estado vital postquirúrgico)
Variables 86-105: Radioterapia primer y último esquema
Variables 106-110: Trasplante de células progenitoras hematopoyéticas
Variables 111-124: Tratamiento complementario (reconstructiva, paliativo, nutrición, rehabilitación)
Variables 125-134: Situación actual, novedades administrativas y clínicas, estado vital
⚠ COMODINES CRÍTICOS: 1800-01-01 (desconocido), 1845-01-01 (No Aplica), 1846-01-01 (ente territorial). El motor debe diferenciarlos correctamente para evitar rechazos del aplicativo SISCAC.
REGLAS CRUZADAS: La variable 128 (Novedad Administrativa) determina la lógica de validación de múltiples campos. Por ejemplo, novedad 4 (fallecido) requiere fecha de muerte (V131) y causa (V132). La variable 18 (fecha diagnóstico) debe coincidir con V24 cuando el diagnóstico es histopatológico.

CREATE TABLE eapb (

id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

// lib/validations/engine.ts
interface ValidationRule {
variable: number;
name: string;
type: 'format' | 'range' | 'required' | 'cross' | 'business';
validate: (value: string, row: CACRecord, context: ValidationContext) => ValidationResult;
suggestion?: (value: string, row: CACRecord) => string;
}
interface ValidationContext {
catalogos: {
cie10: Set<string>;
atc: Set<string>;
cups: Set<string>;
divipola: Set<string>;
ips: Set<string>;
};
allRows?: CACRecord[];  // para validaciones de duplicados
}
// lib/validations/rules/v17-diagnostico.ts
export const rulesV17: ValidationRule[] = [
{
variable: 17,
name: 'cie10_valido',
type: 'range',
validate: (value, row, ctx) => {
if (!ctx.catalogos.cie10.has(value)) {
return { valid: false, error: 'CIE-10 no existe en catálogo CAC' };
}
return { valid: true };
},
suggestion: (value) => `Buscar en catálogo operativo CIE-10 CAC: ${value}`
},
{
variable: 17,
name: 'insitu_estadio_coherente',
type: 'cross',
validate: (value, row) => {
const isInsitu = value.startsWith('D');
const estadio = parseInt(row.v29_estadificacion);
if (isInsitu && estadio !== 0) {
return { valid: false, error: 'CIE-10 "D" (in situ) pero estadio ≠ 0' };
}
return { valid: true };
}
}
];
